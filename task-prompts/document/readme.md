Generate a README for `$PROJECT`.

Inspect the repository structure, entrypoints, package or build manifests, tests, and deployment or environment configuration before writing.

The README should include:

- purpose and architecture summary
- setup and run commands
- test and validation commands
- key directories and notable files
- contribution or extension guidance if the repo supports it

Keep it concrete and derived from the codebase, not marketing language.
