name: adr
description: Write an Architecture Decision Record for a specific technical decision.
category: document
---

Write an ADR for `$DECISION`.

Read the relevant implementation, surrounding architecture, and existing docs or issues first.

Use this structure:

- Title
- Status
- Context
- Decision
- Alternatives Considered
- Consequences

Constraints:

- explain tradeoffs, not just the chosen option
- tie the decision to concrete system constraints
